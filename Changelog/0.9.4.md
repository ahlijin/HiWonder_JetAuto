# 0.9.4

## 更新内容

### 消息包冲突修复

- **删除了冲突的 `servo_controller_msgs` 包**：该包与 `ros_robot_controller_msgs` 定义了同名消息（`ServoPosition`、`ServosPosition`），导致 rosidl C 扩展在 Python 类型注册时断言失败
- **`ServoState.msg` / `ServoStateList.msg` 移至 `ros_robot_controller_msgs`**：原 `servo_controller_msgs` 独有消息，现归入统一的硬件接口消息包
- **`ServosPosition.msg` 新增 `string position_unit` 字段**：补齐原 `servo_controller_msgs` 中的字段定义
- **`ServoPosition.msg` 的 `position` 字段类型统一为 `float32`**：与原有使用方式兼容

### 引用更新

- 更新 Pi3 侧所有旧 `servo_controller_msgs` 引用为 `ros_robot_controller_msgs`：
  - `controller_manager.py`（同时将本地类 `ServoState` 重命名为 `ServoStateData` 避免冲突）
  - `bus_servo_control.py`
  - `init_pose.py`
  - `joystick_control.py`
- 更新 Jetson 侧引用：
  - `servo_control.py` import 更新
  - `jetauto_peripherals/package.xml` 依赖从 `servo_controller_msgs` → `ros_robot_controller_msgs`

### 启动蜂鸣修复

- `startup_check.py` 不再使用固定 `time.sleep(2.0)`，改为等待 `ros_robot_controller/init_finish` 服务就绪后再蜂鸣（超时 10 秒，超时则跳过并打 warning）
- 解决了 Pi3 启动后无蜂鸣提示音的问题

### 键盘控制优化

- **修复 Jetson 侧 `teleop_key_control.py` 按键松开后小车不停止的问题**：按下按键移动，松开按键自动发布零速 `Twist` 停止小车，无需额外按键

### 编译说明变更

Pi3：

```bash
cd ~/pi3_ws
rm -rf build/servo_controller_msgs install/servo_controller_msgs
rm -rf build/ros_robot_controller_msgs install/ros_robot_controller_msgs
colcon build --packages-select ros_robot_controller_msgs
colcon build
```

Jetson：

```bash
cd ~/jetson_ws
rm -rf build/servo_controller_msgs install/servo_controller_msgs
rm -rf build/ros_robot_controller_msgs install/ros_robot_controller_msgs
colcon build --packages-select ros_robot_controller_msgs jetauto_peripherals slam lidar_receiver
```
